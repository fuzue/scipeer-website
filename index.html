Under the graph.

Scipeer is a reasoning layer, not another tool. It connects to where your work already lives and makes every AI session start fully informed.

Graph store

PostgreSQL + pgvector. Every entity, relationship, and provenance record is queryable. Semantic search over embeddings, exact query over structure.

  • Typed nodes with full provenance (actor, timestamp, reason)
  • Versioned entities, data files, notebooks, hypotheses
  • Semantic similarity search at query time

MCP tools for Claude

Scipeer exposes 9 MCP tools. When you open Claude Code in your project directory, it already has the full graph in context.

get_context add_experiment add_hypothesis link_literature search_graph propose_action get_timeline export_report get_provenance

Connectors

Scipeer ingests from wherever your research lives. Domain-agnostic, works for wet labs, AI research, physics, computational biology.

  • Any LIMS, openBIS, SampleDB, Chemotion, Benchling, Labguru
  • Jupyter notebooks with executed outputs
  • Semantic Scholar + PubMed literature pipeline
  • Raw instrument files, FCS, HDF5, CSV
  • REST APIs and generic CSV connectors

Staged autonomy

The agent reasons and proposes, it cannot act. Every action that touches the physical world produces a proposal node. A human approves or rejects. That decision is recorded.

  • Proposal → Review → Approve/Reject → Execute
  • Approval recorded as a typed edge with timestamp
  • No silent writes to external systems in v1
